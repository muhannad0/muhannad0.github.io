Introduction

In the first part of this guide, I’m going to explain how I’m deploying this website to Github Pages with couple of git commit (and followed by git push of course).

I’ll link to the other guides I used for reference as we go along the process.

I’m using Hugo as a static site generator because:

  • It is very quick to build.
  • The ability to use markdown syntax to create your pages is nice.
  • There’s a nice selection of themes to use on your website.

Next part I’ll show how I used the new Github Actions feature to automate publishing it to Github Pages.

Preparation

Install Hugo, create your website

I won’t go into much detail about how to install Hugo and create a website. Their Quick Start has pretty much got you covered.

Ready the Git repos

Following Hugo’s documentation to setup a website on Github Pages, I’ve got the following repos setup on my dev machine:

  • A private repo my-website for the source files.
  • A public repo <username>.github.io to host on Github pages.

Some Tips:

  • Use .gitignore to ignore themes and build files/folders. Use .gitkeep to keep required folders that are initially empty when you start. You can have a look at the file I’m using here.

Manual First Run

You should try a manual deploy to make sure everything works fine before we move forward.

Conclusion

With this you can now use Hugo to create a simple static website. You have also setup the necessary git repos to store your source files and deploy the built website to Github Pages.

In the next part, we’ll use Github Actions to deploy the website automatically on a commit to the source files.
